# 学习者画像
更新时间：2026-04-19

## 学习风格
- 偏好解释模式：先理解底层机制，再应用
- 节奏：中速，需要时间消化抽象概念
- 回应最佳方式：追问式（predict + debug 类问题）
- 挣扎领域：从"理解单个模式"到"系统性设计"的跨越，倾向于碎片化记忆而非结构化组织

## 误区模式
- 倾向于过度套用 Commitment 框架（已出现 2 次），将所有约束机制归结为"承诺/自洽"
- 倾向于统一归因（跳步/找借口/假完成都归到"流程问题"），未区分不同失败动机对应不同防御层
- 在修订练习中会丢弃之前版本的优点，说明模式尚未内化为设计直觉
