# 技能设计中的说服原则

## 概述

大语言模型（LLM）与人类一样，会对相同的说服原则产生响应。理解这一心理学原理有助于你设计更有效的技能——目的不是操控，而是确保关键实践即使在压力下也能被遵循。

**研究基础：** Meincke 等人（2025）在 N=28,000 次 AI 对话中测试了 7 项说服原则。说服技术使合规率提高了一倍以上（33% → 72%，p < .001）。

## 七项原则

### 1. 权威
**定义：** 对专业知识、资质或官方来源的遵从。

**在技能中的作用机制：**
- 祈使语态："你必须（YOU MUST）"、"绝不（Never）"、"始终（Always）"
- 不可协商的措辞："毫无例外（No exceptions）"
- 消除决策疲劳和自我辩解

**适用场景：**
- 纪律执行类技能（TDD、验证要求）
- 安全关键实践
- 既定的最佳实践

**示例：**
```markdown
✅ 先写代码再写测试？删掉。从头开始。毫无例外。
❌ 在可行的情况下考虑先编写测试。
```

### 2. 承诺
**定义：** 与先前的行为、声明或公开表态保持一致。

**在技能中的作用机制：**
- 要求声明："宣布技能使用"
- 强制明确选择："选择 A、B 或 C"
- 使用追踪：用 TodoWrite 管理清单

**适用场景：**
- 确保技能被实际遵循
- 多步骤流程
- 问责机制

**示例：**
```markdown
✅ 当你找到技能时，你必须宣布："我正在使用 [技能名称]"
❌ 可以考虑让你的搭档知道你正在使用哪个技能。
```

### 3. 稀缺
**定义：** 因时间限制或有限可用性而产生的紧迫感。

**在技能中的作用机制：**
- 有时限的要求："在继续之前"
- 顺序依赖："在 X 之后立即"
- 防止拖延

**适用场景：**
- 即时验证要求
- 时间敏感的工作流
- 防止"以后再做"

**示例：**
```markdown
✅ 完成任务后，立即请求代码审查，然后再继续。
❌ 你可以在方便的时候审查代码。
```

### 4. 社会认同
**定义：** 顺从他人的做法或被视为常态的行为。

**在技能中的作用机制：**
- 普遍模式："每次（Every time）"、"始终（Always）"
- 失败模式："没有 Y 的 X = 失败"
- 建立规范

**适用场景：**
- 记录普遍性实践
- 警告常见失败
- 强化标准

**示例：**
```markdown
✅ 没有 TodoWrite 追踪的清单 = 步骤会被跳过。每次都是如此。
❌ 有些人发现 TodoWrite 对管理清单有帮助。
```

### 5. 认同感
**定义：** 共同身份、"我们感"、群体归属。

**在技能中的作用机制：**
- 协作语言："我们的代码库"、"我们是同事"
- 共同目标："我们都追求质量"

**适用场景：**
- 协作式工作流
- 建立团队文化
- 非层级式实践

**示例：**
```markdown
✅ 我们是一起工作的同事。我需要你坦诚的技术判断。
❌ 如果我错了，你大概应该告诉我。
```

### 6. 互惠
**定义：** 回报所受好处的义务感。

**作用机制：**
- 谨慎使用——可能产生操控感
- 在技能中很少需要

**何时避免：**
- 几乎总是（其他原则更有效）

### 7. 喜好
**定义：** 偏好与我们喜欢的人合作。

**作用机制：**
- **不要用于合规目的**
- 与坦诚反馈文化相冲突
- 会制造谄媚行为

**何时避免：**
- 在纪律执行场景中始终避免

## 按技能类型组合原则

| 技能类型 | 使用 | 避免 |
|----------|------|------|
| 纪律执行 | 权威 + 承诺 + 社会认同 | 喜好、互惠 |
| 指导/技术 | 适度权威 + 认同感 | 过度权威 |
| 协作式 | 认同感 + 承诺 | 权威、喜好 |
| 参考资料 | 仅清晰表达 | 所有说服原则 |

## 为什么有效：心理学原理

**明确规则减少自我辩解：**
- "你必须"消除了决策疲劳
- 绝对化语言消除了"这是例外吗？"的疑问
- 明确的反辩解措辞堵住特定漏洞

**执行意图创造自动行为：**
- 清晰的触发条件 + 必须的行动 = 自动执行
- "当 X 发生时，做 Y"比"一般性地做 Y"更有效
- 降低合规的认知负荷

**LLM 具有类人特性：**
- 在包含这些模式的人类文本上训练
- 权威语言在训练数据中先于合规行为出现
- 承诺序列（声明 → 行动）被频繁建模
- 社会认同模式（每个人都做 X）建立规范

## 道德使用

**正当使用：**
- 确保关键实践被遵循
- 创建有效的文档
- 防止可预测的失败

**不正当使用：**
- 为个人利益进行操控
- 制造虚假紧迫感
- 基于内疚的合规

**检验标准：** 如果用户完全理解这项技术，它是否仍服务于用户的真实利益？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 七项说服原则
- 影力研究的实证基础

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 在 N=28,000 次 LLM 对话中测试了 7 项原则
- 合规率从 33% 提升至 72%
- 权威、承诺、稀缺最为有效
- 验证了 LLM 行为的类人模型

## 快速参考

在设计技能时，请自问：

1. **这是什么类型？**（纪律执行 vs. 指导 vs. 参考资料）
2. **我试图改变什么行为？**
3. **哪些原则适用？**（纪律执行通常是权威 + 承诺）
4. **我组合了太多吗？**（不要同时使用全部七项）
5. **这合乎道德吗？**（是否服务于用户的真实利益？）
