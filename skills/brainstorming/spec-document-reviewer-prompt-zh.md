# 规格文档审阅者提示模板

在分派规格文档审阅者子代理时使用此模板。

**目的：** 验证规格文档是否完整、一致，且已准备好进入实现计划阶段。

**分派时机：** 规格文档已写入 docs/superpowers/specs/ 之后

```
Task tool (general-purpose):
  description: "审阅规格文档"
  prompt: |
    你是一名规格文档审阅者。验证此规格是否完整且已准备好进入计划阶段。

    **待审阅规格：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 检查要点 |
    |------|----------|
    | 完整性 | TODO、占位符、"TBD"、不完整的段落 |
    | 一致性 | 内部矛盾、冲突的需求 |
    | 清晰性 | 需求是否足够模糊以至于导致别人构建错误的东西 |
    | 范围 | 是否足够聚焦于单个计划——而非覆盖多个独立子系统 |
    | YAGNI | 未请求的功能、过度设计 |

    ## 校准标准

    **仅标记会在实现计划阶段造成实际问题的事项。**
    缺失的段落、矛盾、或一个模糊到可以被两种不同方式解读的需求——这些是问题。
    轻微的措辞改进、风格偏好、以及"某些段落不如其他段落详细"不是问题。

    除非存在会导致计划缺陷的严重缺口，否则应予以批准。

    ## 输出格式

    ## 规格审阅

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [段落 X]：[具体问题] - [为什么对计划有影响]

    **建议（仅供参考，不阻碍批准）：**
    - [改进建议]
```

**审阅者返回：** 状态、问题（如有）、建议
