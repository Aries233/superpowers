Architecture map

Superpowers 设计地图

这张图把 Superpowers 拆成三层：左侧是启动和适配系统，中间是开发主链路，右侧是质量与贡献门禁。学习时先看主链路，再看两侧支撑系统。

bootstrap

启动与适配层

这一层决定 agent 是否能在正确时机知道“我应该使用技能”。没有 bootstrap，skills 只是磁盘上的文档。

using-superpowers：技能系统入口 GEMINI.md：Gemini 环境入口 OpenCode plugin：注册技能与工具映射 session-start hook：会话启动注入 Codex 同步脚本
main workflow

开发主链路

1. Brainstorming 在写代码前澄清目标、约束、成功标准，并把想法变成设计。 2. Writing Plans 把已批准设计拆成足够具体的小任务、文件路径和验证步骤。 3. Using Git Worktrees 为实现创建隔离工作区，降低污染主线和误伤用户改动的风险。 4. Test-Driven Development 通过红绿重构让实现建立在可见证据上，而不是完成声明上。 5. Subagent-Driven Development 用子代理执行任务，再分别做规格符合性和代码质量审查。 6. Finish Branch 完成所有任务后验证状态，并让用户选择合并、PR、保留或丢弃。
quality gates

质量与贡献门禁

这一层保护项目和 human partner，防止 agent 为了显得主动而提交低质量、无证据或不属于 core 的改动。

CLAUDE.md：Claude Code 贡献规则 AGENTS.md：通用 agent 贡献规则 requesting-code-review：审查门 verification-before-completion：完成前验证 docs/testing.md：真实会话测试
transfer

最值得迁移

迁移“先设计、再计划、再验证”的流程链；迁移强门禁；迁移完成必须有证据的原则。

查看最小迁移方案
